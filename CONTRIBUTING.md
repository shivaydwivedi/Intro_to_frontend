# Contributing

Thank you for helping improve this beginner-friendly frontend learning repository.

## Good Contributions

Helpful contributions include:

- Clearer project instructions
- Accessibility improvements
- Responsive layout fixes
- Beginner-friendly explanations
- Bug reports with reproduction steps
- Small code improvements that keep the projects easy to understand

## Before You Start

1. Read the root `README.md`.
2. Read `ROADMAP.md`.
3. Read the relevant files in `docs/`.
4. Check whether an issue already exists for your idea.

## Project Guidelines

- Use vanilla HTML, CSS, and JavaScript.
- Do not add frameworks or build tools.
- Keep starter and solution code separate.
- Keep code readable for beginners.
- Prefer semantic HTML.
- Include accessible labels, keyboard support, and visible focus states.
- Test responsive behavior before submitting.

## Pull Requests

Before opening a pull request:

- Confirm the project still works in a browser.
- Check changed Markdown links.
- Review the accessibility checklist.
- Keep the pull request focused on one project or one documentation improvement.
- Explain what changed and how it was tested.

## Reporting Issues

When reporting a bug, include:

- The project or file affected
- Steps to reproduce the issue
- What you expected to happen
- What actually happened
- Browser and operating system, when relevant

For learning feedback, describe what felt confusing and where you got stuck.
