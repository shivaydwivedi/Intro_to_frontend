# Project Standards

Each project should follow a consistent structure so learners know where to find instructions, starter code, and completed reference code.

## Standard Project Structure

```text
projects/NN-project-name/
+-- README.md
+-- preview.png
+-- starter/
|   +-- index.html
|   +-- css/
|   |   +-- style.css
|   +-- js/
|   |   +-- app.js
|   +-- assets/
+-- solution/
    +-- index.html
    +-- css/
    |   +-- style.css
    +-- js/
    |   +-- app.js
    +-- assets/
```

Projects that do not require JavaScript may omit the `js/` directory.

Do not add fake screenshots. Add `preview.png` only when a real project preview is available.

## Project README Requirements

Each project README must include:

- Overview
- Preview
- Learning objectives
- Prerequisites
- Requirements
- Accessibility requirements
- Suggested workflow
- Running instructions
- Project structure
- Testing checklist
- Bonus challenges
- Helpful documentation links
- Solution guidance

## Starter Code

Starter code should:

- Give learners enough structure to begin.
- Leave the core learning work for the learner.
- Avoid hidden dependencies.
- Include comments only where they clarify the task.
- Use clear file and folder names.

## Solution Code

Solution code should:

- Meet the project requirements.
- Stay readable for beginners.
- Use semantic HTML, modern CSS, and vanilla JavaScript.
- Include responsive behavior.
- Include accessible labels, states, and focus styles.
- Include loading, empty, success, and error states when applicable.

## Review Checklist

Before marking a project Complete:

- The starter and solution folders are separate.
- The project README is complete.
- The solution meets the stated requirements.
- The page is responsive.
- Keyboard navigation works.
- Focus states are visible.
- Browser console errors are resolved.
- Root README and ROADMAP status are updated.
