# Codex Repository Instructions

These instructions apply to all future Codex sessions in this repository.

## Required Context Before Editing

Before making changes, Codex must read:

- `AGENTS.md`
- `README.md`
- `ROADMAP.md`
- Relevant files in `docs/`
- The README for the specific project being edited, when it exists

Codex must also inspect Git status and preserve unrelated user changes.

## Scope Control

- Work only on the project explicitly requested by the user.
- Do not modify completed projects unless the user explicitly authorizes it.
- Keep starter code separate from completed solution code.
- Do not create or implement future projects early.
- Update the root `README.md` and `ROADMAP.md` when project status changes.

## Technical Standards

- Use semantic HTML, modern CSS, and vanilla JavaScript.
- Avoid frameworks, build tools, TypeScript, and unnecessary dependencies unless the user explicitly approves them.
- Prefer understandable beginner-level code over clever abstractions.
- Keep naming clear and consistent.
- Add comments only when they help beginners understand non-obvious logic.

## Accessibility and UX Standards

- Create responsive and accessible interfaces.
- Support keyboard navigation.
- Provide visible focus states.
- Use meaningful text alternatives for informative images.
- Respect `prefers-reduced-motion` when animations or transitions are used.
- Include loading, empty, success, and error states when applicable.
- Do not rely on color alone to communicate important state.

## Project Documentation

Every project README must eventually include:

- Overview
- Preview
- Learning objectives
- Prerequisites
- Requirements
- Accessibility requirements
- Suggested workflow
- Running instructions
- Project structure
- Testing checklist
- Bonus challenges
- Helpful documentation links
- Solution guidance

## Verification

Before reporting completion, Codex must verify:

- Functionality works for the requested project.
- Layout is responsive.
- Keyboard navigation is usable.
- Focus states are visible.
- Markdown links added or changed still point to valid local files.

## Git Safety

- Never commit, push, delete branches, or rewrite Git history unless explicitly requested.
- Do not run destructive Git commands.
- Do not revert unrelated user changes.
- Report files changed, tests performed, results, assumptions, and limitations.
