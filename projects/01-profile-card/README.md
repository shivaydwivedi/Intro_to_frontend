# Project 01: Profile Card

Build a polished personal profile card with semantic HTML and beginner-friendly CSS.

## Overview

This first project focuses on turning structured content into a small, responsive card. You will practice HTML landmarks, headings, links, lists, the CSS box model, typography, colors, spacing, and simple Flexbox alignment.

## Final Result

The completed card includes a fictional profile image substitute, name, role, short biography, location, frontend skills, portfolio, GitHub, email links, and one primary call to action.

No JavaScript is needed for this project.

## Learning Objectives

- Write semantic HTML for a small profile component.
- Use headings, paragraphs, lists, and links correctly.
- Apply the CSS box model with padding, borders, margin, and sizing.
- Create visual hierarchy with type, color, and spacing.
- Use Flexbox for simple alignment and wrapping.
- Make an image or visual substitute responsive.
- Design a card that works from narrow mobile screens to desktop.
- Add visible focus styles for keyboard users.

## Prerequisites

- Basic HTML tags and attributes
- Basic CSS selectors
- How to open an HTML file in a browser
- Familiarity with browser developer tools is helpful

## Requirements

Your profile card should include:

- A profile image or accessible visual substitute
- A fictional person's name
- A professional role
- A short biography
- A location
- A short list of frontend skills
- Links for portfolio, GitHub, and email
- One clear primary call to action

Use only HTML and CSS. Do not add JavaScript, frameworks, external fonts, icon libraries, or build tools.

## Accessibility Requirements

- Use correct document landmarks.
- Keep a logical heading order.
- Provide meaningful alternative text for the profile visual.
- Use descriptive link text or accessible names.
- Make all links keyboard accessible.
- Add visible focus indicators.
- Keep color contrast readable.
- Do not rely on color alone to communicate meaning.
- Use touch-friendly link sizes.
- Support increased text size without clipping content.

## Suggested Workflow

1. Open `starter/index.html` and read the TODO comments.
2. Finish the missing HTML content first.
3. Open `starter/css/style.css`.
4. Add global styles for the page, type, and links.
5. Style the card with width, padding, border, background, and shadow.
6. Use Flexbox to align the profile header, skills, and links.
7. Test the page at a narrow width such as 320px.
8. Test keyboard focus by pressing Tab through the links.
9. Compare your work with the `solution/` folder only after you have tried the challenge.

## Manual Testing Checklist

- The page opens without JavaScript.
- The profile card content is readable at 320px wide.
- The layout remains comfortable on wider screens.
- There is no horizontal scrolling on narrow screens.
- All links can be reached with the keyboard.
- Focus styles are clearly visible.
- The profile visual has useful alternative text.
- Link text clearly describes each destination.
- Text remains readable when browser zoom is increased.

## Bonus Challenges
